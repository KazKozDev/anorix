# Virtual Friend Architecture

## Архитектура виртуального друга

### Обзор

Virtual Friend построен на базе модульной архитектуры LangChain с расширенными возможностями для создания персонализированного ИИ-компаньона. Система состоит из нескольких ключевых компонентов, которые работают совместно для создания эволюционирующего виртуального друга.

## Основные компоненты

### 1. VirtualFriend (agent/virtual_friend.py)

Главный класс-наследник от `OllamaAgent`, который интегрирует все компоненты виртуального друга.

**Ключевые особенности:**
- Расширенный системный промпт для дружеского общения
- Интеграция всех инструментов друга
- Пре- и пост-обработка взаимодействий
- Автоматическое обновление статистики и адаптация

### 2. RAG-based Personal Facts (rag_management / rag_retrieval)

Все факты о пользователе теперь сохраняются и извлекаются через векторное хранилище (RAG). Для сохранения используется `rag_management` (действие `add_text` с метаданными `{"type":"user_fact", ...}`), а для извлечения — `rag_retrieval` с фильтром `{"type":"user_fact"}`.

**Примеры:**
- Сохранить: `rag_management(action='add_text', content='Меня зовут Артём', metadata={"type":"user_fact","field":"name"})`
- Поиск: `rag_retrieval(query='имя', k=3, filter_metadata={"type":"user_fact"})`

### 3. Emotional Intelligence Tool (agent/tools/emotional_intelligence.py)

Система эмоционального интеллекта для анализа настроения и предоставления поддержки.

**Функции:**
- Анализ эмоций по ключевым словам и паттернам
- История настроений пользователя
- Адаптивные поддерживающие ответы
- Отслеживание эмоциональных трендов

**Поддерживаемые эмоции:**
- happy, sad, angry, anxious, excited, tired, confused, grateful, lonely, confident

### 4. Personality System (agent/tools/personality_system.py)

Система личности друга с возможностью адаптации и эволюции.

**8 черт характера:**
- empathy (эмпатия) - чуткость к эмоциям пользователя
- humor (юмор) - склонность к шуткам
- curiosity (любопытство) - интерес к жизни пользователя
- supportiveness (поддержка) - готовность помочь
- playfulness (игривость) - легкость в общении
- wisdom (мудрость) - склонность давать советы
- energy (энергичность) - живость в общении  
- loyalty (лояльность) - преданность дружбе

**4 стиля общения:**
- empathetic - чуткий и понимающий
- playful - игривый и веселый
- wise - мудрый и рассудительный
- energetic - энергичный и живой

### 5. Proactive Care (agent/tools/proactive_care.py)

Система проактивной заботы о пользователе.

**Функции:**
- Настраиваемые напоминания с повторением
- Проверки самочувствия и советы по здоровью
- Ободряющие сообщения
- Автоматические дружеские проверки

### 6. Enhanced Tool Manager (agent/tool_manager.py)

Расширенный менеджер инструментов, интегрирующий все компоненты друга с существующими инструментами.

## Поток данных

```
Пользователь -> VirtualFriend.process_query()
    |
    v
1. Pre-processing:
   - Обновление статистики взаимодействий
   - Анализ эмоций в запросе
    |
    v
2. Main processing:
   - Обработка через LangChain агента
   - Использование всех доступных инструментов
   - Генерация ответа
    |
    v  
3. Post-processing:
   - Адаптация личности на основе взаимодействия
   - Проверка необходимости проактивной заботы
    |
    v
Ответ пользователю
```

```

```

## Интерфейсы

### 1. Friend Interface (friend_interface.py)

Интерактивный CLI интерфейс с командами для управления всеми аспектами друга.

### 2. Demo Script (demo_friend.py)

Демонстрационный скрипт для показа возможностей виртуального друга.

### 3. Programmatic API

```python
from agent import VirtualFriend

friend = VirtualFriend(friend_name="Anorix")

# Основное общение
response = friend.process_query("Привет!")

# Прямая работа с компонентами
friend.remember_about_user("Любит кофе", "preference")
friend.analyze_user_mood("Сегодня отличный день!")
friend.adapt_personality("Пользователь любит юмор")

# Проверка статуса
status = friend.get_friend_status()
```

## Эволюция и адаптация

Виртуальный друг эволюционирует через:

1. **Адаптацию личности** - черты характера меняются в зависимости от стиля общения пользователя
2. **Накопление памяти** - постоянное изучение пользователя и запоминание деталей
3. **Эмоциональную настройку** - адаптация поддержки под эмоциональные потребности
4. **Проактивную заботу** - развитие системы напоминаний и заботы

## Расширяемость

Архитектура позволяет легко добавлять новые возможности:

1. **Новые инструменты** - добавление в `tool_manager.py`
2. **Новые черты личности** - расширение `personality_system.py`
3. **Новые эмоции** - дополнение паттернов в `emotional_intelligence.py`
4. **Новые типы фактов** - добавление новых метаданных и политик извлечения в RAG

## Безопасность и конфиденциальность

- Все данные хранятся локально
- Нет передачи личных данных на внешние серверы
- Контроль над всеми компонентами системы
- Возможность полного удаления данных пользователя
